# 概要

Azure 向け ASP.NET Core アプリの開発プロセスは、ローカル開発、ソース管理、CI、CD、Azure ホスティング、監視までを一続きで考えます。

アプリはローカルで実装し、自動テストを通し、共有リポジトリへ push し、CI でビルドとテストを行い、CD で Azure へデプロイします。

```mermaid
flowchart LR
    Local["Local dev"] --> Git["Git repository"]
    Git --> CI["CI\nBuild + Test"]
    CI --> Artifact["Artifact"]
    Artifact --> CD["CD"]
    CD --> Azure["Azure App Service"]
    Azure --> Monitor["Monitor / Diagnose"]
```

この流れを早めに整えると、環境差分や手作業デプロイによる事故を減らせます。
