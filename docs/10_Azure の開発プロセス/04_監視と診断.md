# 監視と診断

デプロイして終わりではなく、動いているアプリを観測できることが重要です。

Azure では Application Insights や Azure Monitor を使って、request、例外、依存先呼び出し、パフォーマンス、ログ、可用性を確認できます。

新しく計測を入れる場合は、Application Insights の従来 SDK だけでなく、Azure Monitor OpenTelemetry Distro も確認します。OpenTelemetry は trace、metrics、logs を標準的な形で扱うため、将来の監視基盤の変更にも対応しやすくなります。

監視で見るべき基本は次の通りです。

| 観点 | 例 |
| --- | --- |
| 可用性 | health check、応答成功率 |
| 性能 | p95 latency、依存先の遅延 |
| エラー | 例外数、HTTP 5xx / 4xx |
| リソース | CPU、メモリ、接続数 |
| ユーザー影響 | 失敗した操作、遅い画面 |

ログは構造化しておくと検索しやすくなります。相関 ID を使うと、1 つの request がどの処理を通ったか追跡できます。

運用設計では、アラートを増やしすぎないことも大切です。対応すべき異常に絞り、原因調査に必要な情報を残します。
