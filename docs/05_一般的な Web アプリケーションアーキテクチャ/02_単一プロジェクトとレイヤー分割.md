# 単一プロジェクトとレイヤー分割

小さな ASP.NET Core アプリでは、1 つの Web プロジェクトに Controller、View、Service、Entity、DbContext を置く構成でも始められます。

単一プロジェクトの利点は、構造が分かりやすく、参照設定やプロジェクト分割に悩まないことです。学習用、プロトタイプ、小規模な管理画面では十分なことがあります。

ただし、アプリが大きくなると、責務が混ざりやすくなります。その場合はフォルダやプロジェクトでレイヤーを分けます。

| レイヤー | 主な責務 |
| --- | --- |
| Web | HTTP、routing、model binding、view |
| Application | ユースケース、トランザクション境界 |
| Domain | 業務ルール、Entity、Value Object |
| Infrastructure | DB、外部 API、ファイル、メール |

レイヤー分割で大事なのは、名前より依存方向です。Web から Infrastructure を直接使いすぎると、結局 Controller がインフラ詳細を知ることになります。
