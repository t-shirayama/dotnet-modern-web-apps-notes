# 概要

最新の Web アプリケーションは、いつでも利用でき、さまざまな端末からアクセスされ、需要の変化に耐え、セキュリティと変更容易性を保つことが期待されます。

ASP.NET Core は、この前提に合わせて設計されています。クロスプラットフォーム、軽量なホスティング、DI、テスト容易性、クラウドとの相性が主な強みです。

この章では、最新 Web アプリを次の特徴で整理します。

| 特徴 | ASP.NET Core での見方 | 具体例 |
| --- | --- | --- |
| クラウドでホストしやすい | 小さいフットプリントとスケールしやすいホスティング | App Service に載せ、負荷に応じて instance 数を増やす |
| クロスプラットフォーム | Windows / Linux / macOS とコンテナー対応 | macOS で開発し、Linux container で運用する |
| 疎結合 | DI と NuGet による必要な機能だけの組み合わせ | メール送信や DB アクセスを interface 経由にする |
| テスト容易性 | 単体テスト、統合テスト、`TestServer` | 業務ルールは単体テスト、HTTP 経路は機能テストで確認する |
| UI の柔軟性 | MVC、Razor Pages、Web API、SPA、Blazor | 管理画面は Razor Pages、リッチな画面だけ SPA にする |

## このページで覚えること

- 最新 Web アプリの特徴は、クラウド、疎結合、テスト容易性、UI の柔軟性で見る。
- ASP.NET Core は、必要な機能を組み合わせて使えるため、アプリの規模に合わせやすい。
- 「どの技術を使うか」より、「変更しやすく、運用しやすいか」を先に見る。
