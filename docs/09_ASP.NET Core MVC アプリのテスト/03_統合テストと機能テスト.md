# 統合テストと機能テスト

統合テストは、複数の部品を組み合わせて確認します。EF Core と DB、DI 設定、Repository、外部サービスの adapter などが対象です。

機能テストは、HTTP request をアプリに投げ、middleware、routing、model binding、filter、controller、response までを確認します。ASP.NET Core では `WebApplicationFactory` がよく使われます。

```csharp
await using var app = new WebApplicationFactory<Program>();
var client = app.CreateClient();

var response = await client.GetAsync("/catalog");
response.EnsureSuccessStatusCode();
```

機能テストの価値は、実際の request 経路を守れることです。認証が効いているか、validation error が期待通りか、route が壊れていないか、JSON / HTML の契約が変わっていないかを確認できます。

一方で、機能テストは単体テストより遅いので、全分岐をここで確認しようとしない方がよいです。
