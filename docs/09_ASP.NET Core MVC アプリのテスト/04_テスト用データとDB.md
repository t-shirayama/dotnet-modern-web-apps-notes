# テスト用データと DB

データアクセスを含むテストでは、どの DB を使うかが重要です。

EF Core InMemory provider は高速ですが、リレーショナル DB と挙動が違います。SQL の制約、transaction、query translation、case sensitivity などを正しく検証したい場合は、SQLite や実際の DB container を検討します。

| 選択肢 | 向いている用途 |
| --- | --- |
| InMemory | DB の挙動に依存しない軽いテスト |
| SQLite in-memory | リレーショナルに近い軽量テスト |
| Testcontainers / Docker DB | 本番 DB に近い統合テスト |

テストデータは、テスト同士が影響しないようにします。毎回 DB を初期化する、transaction rollback する、テストごとに一意なデータを作るなどの方法があります。

データ準備が複雑すぎるとテストが読みにくくなります。factory や builder を用意し、テストの意図に関係ない値を隠すと保守しやすくなります。
