# 単体テスト

単体テストは、小さな単位を外部依存から切り離して検証します。Domain model、Application Service、validation、計算ロジックなどが主な対象です。

単体テストに向いているコードは、副作用が少なく、入力と出力が明確です。DB、HTTP、時刻、ファイル、乱数、外部 API などは差し替えられるようにします。

```csharp
var order = Order.Create(customerId);
order.AddItem(productId, quantity: 2, unitPrice: 1200);

order.TotalPrice.Should().Be(2400);
```

Controller の単体テストは、場合によっては価値が低くなります。Controller が薄く、Application Service を呼ぶだけなら、HTTP pipeline を通した機能テストの方が重要なことがあります。

単体テストでは、業務ルールの境界値、例外、状態遷移、不正な入力を重点的に確認します。
