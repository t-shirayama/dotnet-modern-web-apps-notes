# 永続化非依存と境界

永続化非依存は、業務ルールをデータベース技術に縛られない形で保つ考え方です。

EF Core は強力ですが、Domain model が `DbContext` や SQL の都合に引きずられると、業務ルールのテストや変更が難しくなります。データ保存のための制約と、業務として正しい状態の制約は分けて考えます。

境界づけられたコンテキストは、同じ言葉でも領域によって意味が違うことを認める考え方です。たとえば「注文」は、購入者画面、倉庫、請求、分析で必要な属性や状態が違うかもしれません。

| 原則 | 実務での問い |
| --- | --- |
| 永続化非依存 | DB なしで業務ルールをテストできるか |
| 境界づけられたコンテキスト | 同じモデルを無理に全領域で共有していないか |
| カプセル化 | 不正な状態を外から作れてしまわないか |

小さなアプリでは、厳密な DDD よりもシンプルな Entity + Service で十分なこともあります。重要なのは、複雑さが増えたときに境界を作れるよう、UI と DB に業務判断を散らさないことです。
