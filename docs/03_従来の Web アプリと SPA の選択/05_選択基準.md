# 選択基準

従来型 Web アプリと SPA の選択では、次の質問を順番に確認します。

| 質問 | 方向性 |
| --- | --- |
| SEO が重要か | 重要なら従来型や SSR を優先 |
| UI 操作が複雑か | 複雑なら SPA / Blazor を検討 |
| API が外部公開されるか | 公開されるなら API 分離が自然 |
| チームの強みはどこか | C# 中心なら MVC / Razor / Blazor、フロント専門なら SPA |
| 認証とセキュリティをどう扱うか | SPA は token / CORS / CSRF の設計が増える |
| デプロイを分けたいか | 分けたいなら SPA + API、単純にしたいなら一体型 |

最初の実装では、最小の複雑さで要件を満たす構成を選ぶのが安全です。管理画面や業務アプリなら Razor Pages / MVC から始め、必要な箇所だけリッチ UI にする方が、学習コストと運用コストを抑えられます。

SPA を採用する場合は、バックエンドを単なる JSON 返却係にしないことも大切です。認可、業務ルール、入力検証、トランザクション境界はサーバー側で守る必要があります。
