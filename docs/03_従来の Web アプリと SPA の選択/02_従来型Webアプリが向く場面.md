# 従来型 Web アプリが向く場面

従来型 Web アプリは、サーバー側で画面を組み立てるモデルです。ASP.NET Core では MVC や Razor Pages が代表です。

次の条件では、従来型が自然です。

- クライアント側の対話が少ない。
- フォーム入力、検索、一覧、詳細、編集が中心である。
- JavaScript が無効でも最低限動かしたい。
- SEO や初回表示の分かりやすさが重要である。
- チームが C# / Razor / サーバー側開発に強い。

従来型の利点は、認証、認可、validation、画面遷移、エラー処理をサーバー側の一貫した流れで扱いやすいことです。フロントエンドとバックエンドを別アプリとして分けないため、ビルドとデプロイも比較的単純です。

一方で、複雑なドラッグアンドドロップ、リアルタイム更新、多段のクライアント状態、オフライン対応などが増えると、サーバー側レンダリングだけでは扱いづらくなります。その場合は部分的な JavaScript、Blazor、SPA の導入を検討します。
