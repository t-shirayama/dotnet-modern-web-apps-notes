# キャッシュとパフォーマンス

キャッシュは、毎回 DB や外部サービスへ問い合わせる必要がないデータを一時保存する仕組みです。

ASP.NET Core では、memory cache、distributed cache、response caching などを使えます。Azure では Azure Cache for Redis を使うこともあります。

キャッシュで大切なのは、何を速くするかより、**古い値をどこまで許容できるか** です。

| データ | キャッシュしやすさ |
| --- | --- |
| 商品カテゴリ、設定値 | しやすい |
| ユーザー権限 | 慎重に扱う |
| 在庫、残高 | 古さが問題になりやすい |
| 個人情報 | セキュリティと削除要件に注意 |

パフォーマンス改善では、まず計測します。N+1、不要な Include、巨大な payload、インデックス不足、同期 I/O、過剰な tracking など、原因を見つけてから対策します。

キャッシュは強力ですが、整合性、失効、監視、障害時の fallback を設計しないと、問題の原因にもなります。
