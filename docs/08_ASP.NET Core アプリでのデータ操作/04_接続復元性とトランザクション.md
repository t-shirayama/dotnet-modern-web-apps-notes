# 接続復元性とトランザクション

クラウド上の DB では、一時的なネットワークエラーやフェールオーバーが起きることがあります。接続復元性は、一時的な失敗に対して retry する仕組みです。

EF Core では provider によって retry strategy を設定できます。Azure SQL などでは、一時的な障害を前提に設計します。

トランザクションは、一連の変更をまとめて成功または失敗にするための境界です。`SaveChanges` は通常 1 回の呼び出し内でトランザクションを扱いますが、複数操作を明示的にまとめたい場合があります。

注意すべきなのは、retry とトランザクション、外部サービス呼び出しを混ぜると難しくなることです。DB 更新後にメール送信やメッセージ送信を行う場合、重複実行や途中失敗を想定します。

| 論点 | 見ること |
| --- | --- |
| Retry | 同じ処理を再実行して安全か |
| Transaction | どこまでを一貫性境界にするか |
| External call | DB と同じ transaction に入らない |
| Idempotency | 重複実行で壊れないか |
