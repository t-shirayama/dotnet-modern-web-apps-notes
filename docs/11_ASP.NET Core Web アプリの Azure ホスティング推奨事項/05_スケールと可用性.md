# スケールと可用性

Azure でのスケールは、インスタンス性能を上げる vertical scale と、インスタンス数を増やす horizontal scale に分けられます。

水平スケールするには、アプリを stateless に近づけます。セッション、ファイル保存、メモリ cache、background job の扱いに注意します。

| 状態 | スケール時の考え方 |
| --- | --- |
| セッション | Cookie / distributed cache |
| ファイル | Blob Storage |
| キャッシュ | Redis / distributed cache |
| Background job | Queue + worker / WebJobs |
| DB | connection pool と query performance |

可用性では、health check、deployment slot、rollback、backup、監視、アラートを用意します。障害は完全には防げないため、検知して戻せる設計が重要です。

コストも設計要素です。常時高い SKU を使うより、スケール条件、キャッシュ、DB query 改善で必要リソースを抑えられることがあります。
