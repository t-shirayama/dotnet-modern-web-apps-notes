# クライアント側技術の選び方

クライアント側技術は、画面の複雑さに合わせて段階的に選びます。

| 要件 | 選択肢 |
| --- | --- |
| 静的表示が中心 | HTML + CSS + Razor |
| 少しだけ動きを付ける | Vanilla JavaScript / 小さな library |
| 既存 jQuery 資産が多い | jQuery を維持しつつ新規複雑機能は分離 |
| 複雑な状態管理 | React / Angular / Vue |
| C# 中心で対話 UI | Blazor |

重要なのは、単純な画面まで SPA に寄せないことです。逆に、複雑な対話 UI を Razor と jQuery の継ぎ足しで作り続けると、状態管理が苦しくなります。

ASP.NET Core 側では、どの UI 技術を使っても、認証、認可、業務ルール、サーバー側 validation を守る必要があります。クライアント側 validation は UX のためであり、信頼境界ではありません。
