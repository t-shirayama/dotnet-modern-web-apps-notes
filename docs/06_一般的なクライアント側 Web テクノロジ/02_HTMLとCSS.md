# HTML と CSS

HTML はページの構造を表し、CSS は見た目とレイアウトを表します。Web アプリの保守性を上げるには、この 2 つを混ぜすぎないことが重要です。

HTML には意味のある要素を使います。`button`、`form`、`label`、`table`、`nav` などを正しく使うと、アクセシビリティ、ブラウザー標準の動作、テストのしやすさが上がります。

CSS は、再利用できる class や layout rule として整理します。インライン style が増えると、変更範囲が追いにくくなります。

| 悪化しやすい状態 | 起きる問題 |
| --- | --- |
| HTML に style が散らばる | デザイン変更が難しい |
| class 名が場当たり的 | 再利用や削除判断が難しい |
| JavaScript が見た目を直接変更 | 状態と表示の関係が追いにくい |

ASP.NET Core MVC / Razor Pages でも、HTML と CSS の基本設計は重要です。サーバー側で生成するからといって、フロントエンドの構造設計が不要になるわけではありません。
