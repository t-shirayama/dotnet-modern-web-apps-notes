# 学習で押さえる軸

このガイド全体で繰り返し出てくる軸は、次の 5 つです。

| 軸 | 問い |
| --- | --- |
| デプロイ単位 | 1 つのアプリでよいか、分割する理由があるか |
| UI の作り方 | サーバー側レンダリング、SPA、Blazor、ハイブリッドのどれが自然か |
| 依存関係 | 業務ロジックがインフラや UI に引きずられていないか |
| データアクセス | EF Core を直接使うか、Repository / Specification で包むか |
| 運用 | CI/CD、監視、ログ、スケールをどこまで自動化するか |

初学時にありがちな落とし穴は、パターン名を先に覚えてしまうことです。Clean Architecture、DDD、Repository、SPA、Container などは便利ですが、目的が曖昧なまま導入すると複雑さだけが増えます。

この備忘録では、各パターンを「導入すると何が楽になり、何が重くなるか」というトレードオフとして整理します。
