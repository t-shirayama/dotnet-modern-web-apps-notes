# このガイドの位置づけ

原典では、ASP.NET Core と Azure によるモノリシック Web アプリケーションのエンドツーエンド設計を扱います。

「モノリス」は悪い言葉として扱われることがありますが、実務では **単一アプリで十分な間は単一アプリの方が強い** 場面が多くあります。サービス分割は、独立したデプロイ、独立したスケール、組織境界、障害分離などの利益がある一方で、通信、監視、データ整合性、認証認可、ローカル開発を難しくします。

このガイドの範囲は、次のように整理できます。

| 扱うこと | 扱わないこと |
| --- | --- |
| ASP.NET Core MVC / Razor / Web API を含む Web アプリ設計 | 大規模マイクロサービス分割の詳細 |
| Azure へのホスティングと CI/CD | Kubernetes 前提の高度な運用設計 |
| Clean Architecture や DDD の導入判断 | すべてのドメインを厳密な DDD にする前提 |
| EF Core とデータアクセス設計 | データ基盤全体の分析設計 |

学習時は「どの技術が優れているか」より、「この条件ならどの構成が過剰で、どの構成が不足か」を考えると役に立ちます。
